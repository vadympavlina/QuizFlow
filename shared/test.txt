# QuizFlow — Pilot Test Package

Це мінімальний пакет для перевірки нової архітектури перед повним розділенням на сторінки.

## Що тут

- `shared/` — спільний код для всіх майбутніх сторінок (залиш у себе в репозиторії у папку `shared/`)
- `tests-new.html` — пілотна версія сторінки "Тести" (переіменована щоб не конфліктувати зі старим `tests.html` чи `index.html`)

## Як задеплоїти

1. Скопіюй папку `shared/` у корінь свого репо QuizFlow
2. Скопіюй `tests-new.html` у корінь
3. Коміть + пуш

Після деплою GitHub Pages (~30-60 сек):
```
https://твій-юзернейм.github.io/твій-репо/tests-new.html
```

## Що перевірити

1. ✅ Сторінка відкривається, видно sidebar зліва
2. ✅ Активний пункт "Тести" підсвічений
3. ✅ Завантаження спінер зникає і видно список тестів
4. ✅ Пошук тестів працює (поле зверху)
5. ✅ Кнопка "Новий тест" відкриває модалку, можна створити тест
6. ✅ Перехід у "Архів" працює (кнопка всередині сторінки)
7. ✅ Всередині архіву є кнопка назад на "Тести" — вона теж працює
8. ✅ Можна редагувати/архівувати/видаляти тести
9. ✅ Колапс sidebar (кнопка стрілки біля логотипу) працює
10. ✅ Ім'я викладача/аватарка внизу sidebar правильні

## Очікувані "не помилки"

- Клік на інших пунктах sidebar (Спроби, Посилання, Журнал, Аналітика, Студенти, Сповіщення, Підозрілі, Онлайн, Новини) → 404. **Це нормально** — бо відповідні html-файли ще не завантажені. Коли переконаєшся що тести працюють — я дам тобі решту файлів

## Якщо є проблеми

Відкрий DevTools (F12) → Console. Скинь скріншот помилок або їх текст.

## Що НЕ чіпати у твоєму репо

- `index.html` — стара версія, продовжує працювати
- `login.html`, `test.html`, `admin.html`, `admin-login.html`, `constructor.html`, `live.html`
- `firebase.json`, `functions/`, `js/` — інфраструктура
